# Summarize PR Failures Skill

This skill automates the analysis of failed Buildkite CI jobs for a GitHub PR. It downloads logs, summarizes failures, posts a comment, and suggests fixes based on the code diff.

## Arguments

- `pr_url` (required): Full GitHub PR URL (e.g., `https://github.com/vllm-project/llm-compressor/pull/2899`)

## Prerequisites

The following environment variables must be set:

- `BUILDKITE_TOKEN` - Buildkite API token for downloading logs
  - Get from: https://buildkite.com/user/api-access-tokens
  - Required scopes: `read_builds`, `read_job_logs`

- `GH_TOKEN` or `GITHUB_TOKEN` - GitHub token for posting comments (usually already configured for `gh` CLI)

Optional:

- `BUILDKITE_ORG` - Buildkite organization slug (defaults to GitHub org name if not set)

## Available Scripts

Scripts are located in `.claude/skills/summarize-pr-failures/scripts/`:

- `download_buildkite_logs.sh` - Download logs from failed Buildkite jobs for a PR
  - Usage: `./download_buildkite_logs.sh <pr_url> <output_dir>`
  - Requires: `BUILDKITE_TOKEN` environment variable
  - Uses `gh` CLI to get PR details and Buildkite API to download logs

- `sanitize_buildkite_logs.py` - Clean and sanitize downloaded logs
  - Usage: `python sanitize_buildkite_logs.py <input_dir> [output_dir] [--aggressive]`
  - Removes ANSI codes, timestamps, progress bars, and decorative elements
  - `--aggressive` flag extracts only error-related content

## Related Skills

- `summarize-log-failures` - Generates structured failure summaries from log files

## Steps to Summarize PR Failures

When this skill is invoked, the following steps will be completed:

1. **Download logs**: Run `download_buildkite_logs.sh` to download logs from failed Buildkite jobs
   - Creates a temporary directory for logs
   - Extracts PR number and HEAD SHA
   - Queries Buildkite API for failed jobs
   - Downloads log files for each failed job

2. **Sanitize logs**: Run `sanitize_buildkite_logs.py` with `--aggressive` flag
   - Removes noise and formatting
   - Extracts only error-relevant content
   - Saves sanitized logs for analysis

3. **Analyze failures**: Invoke `/summarize-log-failures` on the sanitized logs
   - Parses test failures from logs
   - Categorizes and groups failures
   - Generates structured summary

4. **Get code diff**: Fetch the PR diff to understand what changed
   - Use `gh pr diff <pr_number>` to get the full diff
   - Focus on files mentioned in test failures

5. **Suggest fixes**: Analyze failures in context of the code changes
   - Look for obvious issues:
     - Import errors from renamed/moved modules
     - Changed function signatures causing test failures
     - Missing dependencies or configuration
     - Typos in variable/function names
   - For each failure with an obvious fix, provide:
     - File path and line numbers
     - Explanation of the issue
     - Suggested code change (if applicable)

6. **Post comment**: Post the summary as a PR comment using `gh pr comment`
   - Include failure summary
   - Add fix suggestions (if any)
   - Link to Buildkite build
   - Use clear formatting with sections

## Workflow Example

```bash
# Set required environment variables
export BUILDKITE_TOKEN="your_buildkite_token"

# Invoke the skill
/summarize-pr-failures https://github.com/vllm-project/llm-compressor/pull/2899
```

The skill will:
1. Download logs to `.tmp/buildkite-logs-<pr_number>/`
2. Sanitize logs to `.tmp/buildkite-logs-<pr_number>/sanitized/`
3. Generate failure summary
4. Analyze diff and suggest fixes
5. Post comment to PR

## Comment Format

The posted comment will follow this structure:

```markdown
## 🔴 CI Failure Summary

**Build:** [link]
**Commit:** [sha]

### Failed Tests (X)

- `test_module::test_name` - Brief error description
- ...

### Error Analysis

Common patterns identified:
- ...

### Suggested Fixes

<details>
<summary>Fix 1: [Issue description]</summary>

**File:** `path/to/file.py:123`
**Issue:** [Explanation]
**Suggested change:**
\```python
# Suggested code
\```

</details>

---
<sub>Generated by Claude Code</sub>
```

## Notes

- The skill creates temporary directories in `.tmp/` for downloaded logs
- If no jobs failed, the skill will report success and exit early
- If fixes are not obvious from the diff, the comment will only include the failure summary
- For complex failures requiring manual investigation, the full details section provides complete context
